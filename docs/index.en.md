## Documentation Strategy

This repo follows a "run first, optimize later" structure:

1. Get a minimal setup running.
2. Enable advanced capabilities only when needed (retry/failover, context enrichment, anti-drop queue, long-reply forwarding).
3. Harden permissions and anti-abuse settings for production.
